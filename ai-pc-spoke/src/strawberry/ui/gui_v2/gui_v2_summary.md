# Spoke Core GUI V2

The Spoke Core GUI V2 is a modern, responsive, and user-friendly interface for interacting with the Spoke Core. It provides a seamless experience for users to manage their devices, skills, and settings.

## Features

- Modern, responsive, and user-friendly interface
- Seamless experience for users to manage their devices, skills, and settings
- Chat interface and conversation history
- Settings Interface based on the settings schema provided by the SettingsManager

## UI: 
- Chat interface
- Chat Session history (collapsible menu)
- Settings Interface
- Voice Control
    - Transcribe voice toggle (button inside user input text box)
    - Full Voice Mode toggle (button inside user input text box)
- Title Bar
    - Minimize button
    - Maximize button
    - Close button
- Status Bar
    - Device Name
    - Online/Offline status
    - Voice Mode Status

---

# Recommended Interface

## **Layout Option 2: Collapsible Rail** (Recommended)

This layout provides the best balance of:
- **Focus**: Maximum space for chat content
- **Accessibility**: Quick access to navigation via icon rail
- **Discoverability**: Expandable rail reveals full navigation
- **Modern Feel**: Clean, minimal aesthetic similar to modern chat apps

# Component Summary

| Component | Recommendation | Rationale |
|-----------|----------------|-----------|
| Title Bar | Branded | Identity + clean controls |
| Sidebar | Collapsible Rail | Space efficiency + accessibility |
| Chat Area | Card Style | Clear message separation, tool call display |
| Input Area | Floating Bar | Modern feel, integrated voice |
| Status Bar | Bottom Bar | Familiar, consistent |
| Settings | Slide-in Panel | Quick access, maintains context |
| Voice Indicator | Pulsing Orb | Clear, non-distracting |

# Implementation Notes

1. **Frameless Window**: Use `Qt.FramelessWindowHint` with custom title bar
2. **Animations**: Use `QPropertyAnimation` for sidebar expand/collapse
3. **Theme Support**: Leverage existing `theme.py` with CSS variables
4. **Responsive**: Handle window resize gracefully, collapse sidebar on narrow widths
5. **Accessibility**: Keyboard navigation, screen reader support, high contrast mode
6. **Reuse**: Leverage existing widgets from `qt/widgets/` where possible
